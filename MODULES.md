## 一、客户端模块

### 1.1 用户端 (index.html)

**功能概述**：面向普通用户，提供身份绑定、菜单查看、菜品搜索、在线下单、订单查询等功能。

**页面结构**：

| 区域 | 功能描述 |
|------|----------|
| 用户绑定区 | 用户输入用户ID、姓名、工号进行身份绑定 |
| 导航标签 | 今日菜单 / 我的订单 / 历史订单 / 饮食建议 |
| 今日菜单 | 展示上架菜品列表，支持搜索和下单 |
| 我的订单 | 展示今日订单，显示取餐码 |
| 历史订单 | 支持日期范围筛选的历史订单查询 |
| 饮食建议 | 根据今日订单提供热量统计和健康建议 |

**核心流程**：

1. **身份绑定**：用户首次访问 → 输入信息 → 调用 `/api/bind` → 绑定成功后进入主界面
2. **查看菜单**：进入今日菜单 → 调用 `/api/menu` → 展示菜品列表
3. **搜索菜品**：输入关键词 → 调用 `/api/search_dish` → 展示匹配结果
4. **在线下单**：选择菜品和数量 → 调用 `/api/order` → 生成订单和取餐码
5. **查询订单**：进入我的订单 → 调用 `/api/user_orders` → 展示订单列表

**技术特点**：
- 纯前端实现，使用原生 JavaScript
- 支持 localStorage 保存用户登录状态
- 自动取餐提醒（每分钟检查一次）

### 1.2 管理端 (admin.html)

**功能概述**：面向管理员，提供菜品管理、订单管理、系统配置、数据统计等功能。

**页面结构**：

| 区域 | 功能描述 |
|------|----------|
| 侧边导航 | 数据概览 / 菜品管理 / 订单管理 / 系统配置 / 操作日志 |
| 数据概览 | 今日订单统计、热销菜品TOP5、低库存预警 |
| 菜品管理 | 上架新菜品、编辑库存、补货、下架菜品 |
| 订单管理 | 订单核销、取消订单、处理超时订单、订单列表 |
| 系统配置 | 修改下单截止时间、取餐时段 |
| 操作日志 | 查看管理员操作记录（最近20条） |

**核心流程**：

1. **数据概览**：进入首页 → 调用 `/api/admin/stats` → 展示统计数据
2. **菜品上架**：输入菜品信息 → 调用 `/api/admin/add_dish` → 保存并记录日志
3. **订单核销**：输入订单号 → 调用 `/api/verify_order` → 标记为已取餐
4. **取消订单**：输入订单号 → 调用 `/api/admin/cancel_order` → 恢复库存
5. **处理超时**：点击按钮 → 调用 `/api/process_overtime` → 批量处理超时订单

**技术特点**：
- 管理员身份固定为 `admin_ma`（马静）
- 所有操作自动记录到 `admin_logs` 表
- 支持模态框编辑菜品信息

## 三、数据库模块

### 3.1 数据库文件

- **文件位置**：`database/canteen.db`
- **数据库类型**：SQLite（嵌入式文件数据库）
- **运行模式**：WAL（Write-Ahead Logging）

### 3.2 表结构

#### 3.2.1 users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | TEXT | 用户ID（主键） |
| name | TEXT | 用户姓名 |
| employee_id | TEXT | 工号（唯一） |
| role | TEXT | 角色（user/admin） |
| bind_status | INTEGER | 绑定状态（0未绑定/1已绑定） |
| violation_count | INTEGER | 违规次数 |

#### 3.2.2 dishes（菜品表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 自增主键 |
| name | TEXT | 菜品名称（唯一） |
| remaining | INTEGER | 剩余库存 |
| limit_per_person | INTEGER | 每人限购数量 |
| status | TEXT | 状态（上架/下架） |
| initial_stock | INTEGER | 初始库存 |
| nutrition_info | TEXT | 营养信息（JSON格式） |
| allergy_tag | TEXT | 过敏原标签 |
| alias | TEXT | 菜品别名（用于搜索） |

#### 3.2.3 orders（订单表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | 订单号（主键） |
| user_id | TEXT | 用户ID（外键） |
| dish_name | TEXT | 菜品名称 |
| quantity | INTEGER | 数量 |
| order_time | DATETIME | 下单时间 |
| take_deadline | DATETIME | 取餐截止时间 |
| status | TEXT | 状态（pending/taken/cancelled/overtime） |
| pickup_code | TEXT | 取餐码 |

#### 3.2.4 system_config（系统配置表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 自增主键 |
| config_key | TEXT | 配置键（唯一） |
| value | TEXT | 配置值 |
| updated_at | DATETIME | 更新时间 |

#### 3.2.5 admin_logs（管理员操作日志表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 自增主键 |
| admin_id | TEXT | 管理员ID（外键） |
| action | TEXT | 操作类型 |
| detail | TEXT | 操作详情 |
| created_at | DATETIME | 创建时间 |

### 3.3 索引

| 索引名称 | 表名 | 字段 | 说明 |
|----------|------|------|------|
| idx_orders_user_date | orders | user_id, DATE(order_time) | 优化用户订单查询 |
| idx_dishes_name_status | dishes | name, status | 优化菜品搜索 |

### 3.4 初始数据

#### 系统配置

| config_key | value | 说明 |
|------------|-------|------|
| order_cutoff_time | 10:00 | 下单截止时间 |
| take_start | 11:30 | 取餐开始时间 |
| take_end | 12:30 | 取餐结束时间 |

#### 默认管理员

| user_id | name | employee_id | role |
|---------|------|-------------|------|
| admin_ma | 马静 | 0001 | admin |

### 3.5 数据库初始化

**脚本位置**：`database/init_db.py`

**执行方式**：
```bash
python database/init_db.py
```

**自动初始化**：Flask 服务器启动时会自动调用初始化脚本，确保数据库文件存在。

## 五、常见问题

### Q1: 客户端显示"网络错误"怎么办？

**原因**：直接用浏览器打开 HTML 文件（file:// 协议）无法发起 HTTP 请求。

**解决方案**：必须通过 Flask 服务器访问，即访问 http://localhost:5000

### Q2: 管理端所有操作都失败？

**原因**：数据库文件不存在或表结构未初始化。

**解决方案**：
1. 确保 Flask 服务器已启动（会自动初始化数据库）
2. 或手动运行 `python database/init_db.py`

### Q3: 数据库文件在哪里？

**位置**：`database/canteen.db`

**说明**：SQLite 是嵌入式数据库，所有数据存储在单个文件中，无需额外的数据库服务。

### Q4: 如何添加新菜品？

**步骤**：
1. 访问管理端：http://localhost:5000/admin
2. 点击左侧"菜品管理"
3. 填写菜品名称、初始库存、限购数量
4. 点击"上架"按钮

### Q5: 如何测试用户下单？

**步骤**：
1. 访问用户端：http://localhost:5000
2. 输入用户ID（如 test001）、姓名、工号，点击绑定
3. 进入"今日菜单"，选择菜品下单
